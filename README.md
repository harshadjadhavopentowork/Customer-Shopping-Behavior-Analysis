## 🛠️ Data Engineering & Pipeline Breakdown

### 1. Pre-processing & Feature Engineering (Python)

The initial lifecycle utilized `pandas` to clean and normalize the file data:

* **Missing Value Imputation:** Missing `Review Rating` entries were filled dynamically using the median score calculated per product category.


* **Schema Standardization:** Transformed column headers into standardized `snake_case` format.


* **Feature Engineering:**
* Binned continuous customer age distributions into structural categorical classes (`Young Adult`, `Adult`, `Middle-aged`, `Senior`) via quantitative cutting methods.


* Mapped text purchase frequencies into discrete quantitative metrics (`purchase_frequency_days`).




* **Redundancy Optimization:** Deduplicated features by matching matching boolean profiles of `discount_applied` against `promo_code_used` (found 100% correlation) and subsequently dropped the redundant feature.


* **ETL Pipeline:** Configured connection engines via `sqlalchemy` to automatically dump the clean structured frame directly into SQL instances (`PostgreSQL`/`MySQL`/`MS SQL Server`).



### 2. Business Analytics (SQL Queries)

With the data loaded as a relational table named `customer`, a series of complex window functions and aggregating scripts were built to answer corporate problems:

```sql
-- Q1. What is the total revenue generated by male vs. female customers?
SELECT gender, SUM(purchase_amount) AS revenue
FROM customer
GROUP BY gender;

-- Q2. High-Spending Discount Users
SELECT customer_id, purchase_amount 
FROM customer 
WHERE discount_applied = 'Yes' AND purchase_amount >= (SELECT AVG(purchase_amount) FROM customer);

-- Q7. Customer RFM-based Structural Segmentation
WITH customer_type AS (
    SELECT customer_id, previous_purchases,
    CASE 
        WHEN previous_purchases = 1 THEN 'New'
        WHEN previous_purchases BETWEEN 2 AND 10 THEN 'Returning'
        ELSE 'Loyal'
    END AS customer_segment
    FROM customer
)
SELECT customer_segment, COUNT(*) AS "Number of Customers" 
FROM customer_type 
GROUP BY customer_segment;

-- Q8. Top 3 Most Purchased Products Within Each Structural Category
WITH item_counts AS (
    SELECT category, item_purchased, COUNT(customer_id) AS total_orders,
           ROW_NUMBER() OVER (PARTITION BY category ORDER BY COUNT(customer_id) DESC) AS item_rank
    FROM customer
    GROUP BY category, item_purchased
)
SELECT item_rank, category, item_purchased, total_orders
FROM item_counts
WHERE item_rank <= 3;

```

## 🎯 Final Business Recommendations

* 🚀 **Accelerate Subscriptions:** Enhance retention by adding loyalty-tier reward options for low-frequency subscribers.


* 💳 **Optimize Discount Strategies:** Audit discount rules for items showing baseline dependencies over 45% (such as hats and sneakers) to prevent margin erosion.


* 🎯 **Targeted Campaign Sourcing:** Focus higher-cost priority marketing on the highest revenue-generating demographics (Young Adults & Express Delivery users).
